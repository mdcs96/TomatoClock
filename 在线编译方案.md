# 📱 在线编译 Android APK - 无需本地环境

如果你的电脑上没有 Android 开发环境，可以使用这些在线服务免费编译APK：

## 方案一: GitHub Actions (推荐)

### 步骤：

1. **创建 GitHub 仓库**
   - 访问 https://github.com/new
   - 创建新仓库，名称如 `TomatoClock`

2. **上传代码**
   ```bash
   cd ~/Desktop/TomatoClock
   git init
   git add .
   git commit -m "Initial commit: 番茄钟应用"
   git branch -M main
   git remote add origin https://github.com/你的用户名/TomatoClock.git
   git push -u origin main
   ```

3. **创建 GitHub Actions 工作流**

   在仓库中创建文件 `.github/workflows/build.yml`:

   ```yaml
   name: Build Android APK

   on:
     push:
       branches: [ main ]
     workflow_dispatch:

   jobs:
     build:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v3

         - name: Set up JDK 17
           uses: actions/setup-java@v3
           with:
             java-version: '17'
             distribution: 'temurin'

         - name: Grant execute permission for gradlew
           run: chmod +x gradlew

         - name: Build with Gradle
           run: ./gradlew assembleDebug

         - name: Upload APK
           uses: actions/upload-artifact@v3
           with:
             name: TomatoClock-APK
             path: app/build/outputs/apk/debug/app-debug.apk
   ```

4. **下载编译好的 APK**
   - 进入 Actions 标签页
   - 点击最新的工作流运行
   - 下载 Artifacts 中的 APK

**优点**: 完全免费，自动化，支持持续集成

## 方案二: AppVeyor

### 步骤：

1. **访问** https://www.appveyor.com/
2. **登录并关联 GitHub**
3. **添加项目**
4. **配置 appveyor.yml**:

   ```yaml
   version: 1.0.{build}
   image: Ubuntu
   
   install:
     - sudo apt-get update
     - sudo apt-get install -y openjdk-17-jdk
   
   build_script:
     - chmod +x gradlew
     - ./gradlew assembleDebug
   
   artifacts:
     - path: app/build/outputs/apk/debug/*.apk
   ```

5. **触发构建并下载 APK**

**优点**: 免费，配置简单

## 方案三: Gradle Play Publisher (需要 Google Play)

如果你有 Google Play 开发者账号：

1. 配置 Gradle Play Publisher 插件
2. 直接发布到 Google Play
3. 生成 APK 或 AAB

## 方案四: Appetize.io (在线模拟器)

### 步骤：

1. **访问** https://appetize.io/
2. **上传 APK** (需要先编译)
3. **在浏览器中运行和测试**

**优点**: 无需真机，可在线演示

## 方案五: Expo / React Native (重构方案)

如果愿意用 JavaScript 重写：

1. 使用 React Native 或 Flutter Web
2. 使用 Expo Snack 在线开发
3. 直接导出 APK

## 本地快速编译方案

如果想在本地编译，最快的方式：

### macOS / Linux:
```bash
# 1. 安装 Android Studio
brew install --cask android-studio

# 2. 启动 Android Studio，完成初始设置

# 3. 打开项目
# File → Open → 选择 TomatoClock 文件夹

# 4. 等待 Gradle 同步完成

# 5. 编译 APK
# Build → Build Bundle(s) / APK(s) → Build APK(s)
```

**时间**: 首次约15-20分钟（下载SDK和依赖）

## 常见问题

**Q: GitHub Actions 编译失败？**
A: 检查 `gradlew` 文件是否有执行权限，确保 `.github/workflows/build.yml` 格式正确。

**Q: 没有 GitHub 账号怎么办？**
A: 注册免费 GitHub 账号，或使用 Android Studio 本地编译。

**Q: 可以直接下载编译好的 APK 吗？**
A: 出于安全考虑，建议自行编译。未知来源的 APK 可能存在风险。

**Q: 编译时间太长？**
A: 首次编译需要下载依赖（约200MB），后续编译会快很多。
